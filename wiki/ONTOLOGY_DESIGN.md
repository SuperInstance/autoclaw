# Ontology Design & Knowledge Representation

Structuring knowledge formally.

---

## 🏗️ Ontology Components

**Concepts**: Entity types (Person, Document)
**Properties**: Attributes (name, date)
**Relations**: Connections (author, cites)
**Constraints**: Rules (name is string)
**Hierarchies**: Generalization (Animal → Dog)

---

## 📋 Ontology Levels

**Lightweight**: Basic concepts + properties
**Medium**: Constraints + some rules
**Heavy**: Rich semantics + reasoning

---

## 🔄 Design Process

1. Identify key concepts
2. Define properties and relations
3. Build hierarchy
4. Add constraints
5. Test and refine
6. Document

---

## 📊 Representation Languages

**OWL**: Web Ontology Language
**RDF**: Resource Description Framework
**JSON-LD**: JSON Linked Data
**Custom**: Domain-specific formats

---

## 💡 Applications

- Semantic web
- Knowledge graphs
- Enterprise data models
- Domain-specific languages
- Interoperability

---

## 🎯 Design Principles

- **Reusability**: Can use elsewhere
- **Correctness**: Accurate representation
- **Clarity**: Easy to understand
- **Completeness**: Covers domain
- **Consistency**: No contradictions
