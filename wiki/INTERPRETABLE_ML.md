# Interpretable Machine Learning

Building models you can understand and explain.

---

## 🔍 Why Interpretability?

**Trust**: Understand what system does
**Debug**: Find and fix errors
**Compliance**: Audit trails for regulation
**Improvement**: Identify what to change
**Ethics**: Ensure fairness

---

## 📊 Interpretable Models

**Linear Models**: Direct feature weights
**Decision Trees**: If-then rules
**Rule-Based**: Explicit logic
**k-NN**: Similar example comparison
**Naive Bayes**: Conditional probabilities

---

## 📈 Explainability Methods

**Feature Importance**: Which inputs matter?
**LIME**: Local linear approximation
**SHAP**: Game theory-based attribution
**Attention**: What did model focus on?
**Saliency**: Pixel-level importance

---

## ⚖️ Accuracy vs Interpretability

```
Simple model (interpretable)      Complex model (black box)
├─ Decision trees       ├─ Neural networks
├─ Linear models        ├─ Random forests
├─ Rule-based           └─ Gradient boosting
High interpretability   High accuracy
Lower accuracy          Low interpretability
```

---

## 💡 Best Practices

1. Start with simple model
2. Add complexity only if needed
3. Use post-hoc explanation for complex models
4. Validate explanations
5. Involve domain experts
